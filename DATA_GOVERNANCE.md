# LAB MATCH AI 교수·연구실 데이터 운영 원칙

## 수집 목적
학생이 관심 분야와 관련된 교수 및 연구실의 공식 정보를 빠르게 찾고, 공식 홈페이지로 이동할 수 있도록 공개된 직무 정보를 구조화한다.

## 출처 기준
- 공식 대학, 학과, 교수 프로필, 연구실 홈페이지를 우선 사용한다.
- 로그인이나 권한이 필요한 비공개 페이지는 사용하지 않는다.
- 생성형 AI는 추출과 구조화의 보조 도구로만 사용하며 최종 근거는 공식 페이지로 둔다.

## 최소 수집 항목
- 교수 성명
- 소속 학과
- 연구실명
- 공개 연구 분야와 키워드
- 공식 홈페이지 또는 공식 프로필 URL

## 수집하지 않는 항목
- 교수 사진
- 이메일과 전화번호
- 생년월일, 가족관계 등 사적 정보
- 서비스 목적에 필요하지 않은 상세 학력과 개인 경력

## 표현 원칙
- 공식 소개문을 그대로 대량 복제하지 않고 사실 중심 키워드로 요약한다.
- 검색 결과는 키워드 관련도이며 교수나 연구실의 우수성 순위가 아님을 표시한다.
- `최고`, `우수`, `대표 연구실`, `가장 적합한 교수` 같은 평가 표현을 사용하지 않는다.

## 검증 및 변경 관리
- 현재 통합본 기준일은 2026년 7월이다.
- 이 날짜는 전체 교수 정보의 전수 재검증 완료일을 의미하지 않는다.
- 학기별 또는 주요 업데이트 시 소속, 공식 링크와 핵심 연구 분야를 점검한다.
- 오류, 정정 또는 삭제 요청은 GitHub Issues에서 접수하고 공식 자료 확인 후 수정 또는 비노출 처리한다.
- 주요 변경은 Git 커밋 이력으로 남긴다.

## 내부 권장 필드
```json
{
  "source_url": "공식 페이지 URL",
  "source_checked_at": "YYYY-MM-DD",
  "source_type": "official_university_profile | official_lab_page",
  "verification_status": "verified | needs_review",
  "summary_method": "fact_based_paraphrase"
}
```

기존 데이터에 위 필드가 없는 경우 확인되지 않은 값을 임의로 채우지 않고, 향후 검증 시 순차적으로 추가한다.
