# DGIST 핵심 검색 오류 수정 보고서

## 수정 범위

사용자가 지정한 다음 네 항목만 수정했다.

1. 세부 분야 배너 클릭 시 결과 0개가 발생하는 문제
2. `자율주행`처럼 기본적인 직접 검색어가 0개가 되는 문제
3. 좁은 세부 분야에서 관련 없는 교수님이 대량 포함되는 문제
4. 잘리거나 교수명만 반복되는 연구실명 표시 문제

학과 선택 결과에 다른 학과 교수님이 포함되는 동작과 결과 0개 안내 구조는 이번 수정에서 변경하지 않았다.

## 검색 알고리즘 수정

- `ct`, `ai`처럼 짧은 영문 토큰이 다른 단어 내부에서 부분 일치하는 현상을 막았다.
- 문제 세부 분야에는 실제 연구 내용이 확인된 교수님 후보만 남기는 세부 분야별 정제 규칙을 적용했다.
- `CO2 전환·촉매`와 `고분자·지속가능 소재` 배너의 내부 검색어를 실제 DB 표현과 연결되도록 보강했다.
- `자율주행`, `자율주행차`, `autonomous driving`, `autonomous vehicle` 직접 검색에 SLAM, 위치추정, 경로계획, 차량제어 표현을 내부적으로 확장했다.
- 검색어 원문은 외부로 전송하지 않고 기존처럼 브라우저 내부에서만 처리한다.

- `신경인터페이스 및` → `신경인터페이스 및 마이크로시스템 연구실`
- `지능형 이미징 및` → `지능형 이미징 및 비전 시스템 연구실`
- `김경대 교수님` → `지능형 로봇 제어 연구실(ARC Lab)`
- `임성훈 교수님` → `컴퓨터 비전 연구실(CV Lab)`
- `문제일 교수님` → `화학감각신경계 연구실`

## 통계 수집 영향

`analytics.js`와 `analysis/goatcounter_analysis.py`는 수정 전 파일과 SHA-256 해시가 동일하다. 따라서 다음 기존 통계는 그대로 유지된다.

- 페이지뷰
- 학과 및 세부 분야 선택 횟수
- 검색 실행과 결과 수 구간
- 결과 0개 비율
- 공식 홈페이지 및 프로필 클릭
- 더보기와 만족도
- 검색 복구 관련 이벤트

새 개인정보 항목, 검색어 원문, 교수명, 사용자 ID 또는 개인별 이동 경로를 추가로 수집하지 않는다.

## 검증

- DGIST `app.js`, `data.js`, 공통 `analytics.js`, `query-assist.js` JavaScript 구문 검사 통과
- 지정된 8개 검색 및 배너 회귀검사 통과
- 잘린 연구실명 및 교수명 반복 사례 10개 직접 검색 검증 통과
- 기존 통계 파일 미변경 확인
