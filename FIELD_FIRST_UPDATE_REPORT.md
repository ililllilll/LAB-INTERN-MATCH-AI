# 분야 중심 통합 연구실 추천 추가 보고서

## 변경 목적

기존 대학원 연구실 추천은 학교를 먼저 선택한 뒤 해당 학교 안에서만 검색하는 구조였습니다. 이 구조를 유지하면서, 학교를 먼저 정하지 않고 연구 분야를 기준으로 DGIST, 서울대학교, KAIST, POSTECH 연구실을 통합 탐색하는 경로를 추가했습니다.

## 새 서비스 경로

```text
graduate/field/index.html
```

첫 화면은 다음 세 선택지로 구성됩니다.

1. 인턴 추천 서비스
2. 학교별 연구실 추천
3. 분야별 연구실 추천

## 통합 데이터베이스

기존 최종 공개 데이터베이스와 내부 추천용 구조화 데이터를 개인정보 최소화 원칙에 따라 통합했습니다.

| 학교 | 통합 레코드 수 |
|---|---:|
| DGIST | 158 |
| 서울대학교 | 387 |
| KAIST | 447 |
| POSTECH | 236 |
| 합계 | 1,228 |

교수 이메일, 전화번호, 팩스와 사진은 새 통합 데이터에 포함하지 않았습니다.

## 분야 선택 구조

큰 연구 분야 12개와 세부 분야 129개를 구성했습니다.

- 화학
- 물리
- 생명과학
- 뇌과학
- 전기전자
- 컴퓨터공학·AI
- 기계공학
- 화학공학
- 신소재공학
- 에너지·환경
- 수학·통계
- 의생명·바이오융합

로봇과 자율주행, 반도체 공정과 반도체 소자처럼 비슷해 보여도 연구 내용이 다른 분야는 별도 세부 배너로 분리했습니다.

## 결과 카드

각 결과 카드에 다음 정보를 표시합니다.

- 교수명과 영문명
- 연구실명
- 재직 대학
- 소속 학과
- 핵심 연구 분야
- 검색어와 연결되는 매칭 근거
- 주요 연구 방법, 연구 대상과 응용 분야
- 연구실 홈페이지 또는 공식 프로필

검색 결과 순서는 공개 연구 정보와 입력 키워드의 관련도 기준이며 교수 또는 연구실의 우수성 평가가 아닙니다.

## 검색 알고리즘

학교별 기존 내부 추천 DB의 다음 정보를 통합 검색에 사용합니다.

- 주요 및 보조 연구 분야
- 세부 분야
- 연구 방법
- 소재 및 연구 대상
- 응용 분야
- 강한 검색어와 약한 검색어
- 제외 검색어
- 한글 및 영문 동의어
- 화면 표시용 연구 분야와 키워드

같은 키워드가 한 교수 데이터에 반복된다는 이유만으로 과도하게 올라가지 않도록, 검색어별로 가장 강한 근거를 중심으로 점수를 계산하고 여러 독립 근거가 확인될 때만 소폭 가산합니다.

## 통계 반영

공통 `analytics.js`와 Python 분석기에 분야별 통합 서비스 페이지를 추가했습니다.

주요 이벤트:

```text
lm-nav-home-field
lm-lab-major-field-*
lm-lab-banner-field-*
lm-search-outcome-field-*
lm-lab-homepage-field-rank-*
lm-lab-profile-field-rank-*
lm-feedback-field-*
```

검색어 원문은 전송하지 않으며 기존과 동일하게 익명 분야 범주, 검색 직후 표시 결과 수 구간, 공식 링크 클릭과 만족도만 집계합니다.

## 검증

- 통합 데이터 1,228개 확인
- 큰 분야 12개, 세부 분야 129개 확인
- 새 `data.js`, `app.js`, 공통 `analytics.js` JavaScript 문법 검사 통과
- GoatCounter Python 분석기 문법 검사 통과
- 대표 세부 분야 검색 결과 생성 확인
- 교수 카드의 대학 및 학과 표기 확인
- 새 통합 데이터의 이메일, 전화번호, `mailto:`, `tel:` 미포함 확인
- 기존 인턴 및 학교별 연구실 서비스 파일 유지
