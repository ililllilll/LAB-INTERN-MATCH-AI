# LAB MATCH AI 수정 전후 회귀 비교

## 최종 판정

- 교수 및 인턴 레코드 수: 동일
- 비연락처 데이터 구조와 순서: 동일
- 공식 HTTP 및 HTTPS URL 목록과 순서: 동일
- 추천 및 정렬 코드: 동일
- HTML과 UI 코드: 동일
- GoatCounter 및 개인정보 동의 코드: 동일
- 대표 브라우저 검색 출력: 확인한 16개 검색에서 불일치 0건

## 데이터 구조 비교

수정 전 데이터에 연락처 제거 규칙을 적용한 정규화 구조와 수정 후 데이터를 전체 객체 단위로 비교했다.

| 데이터 파일 | 정규화 후 전체 구조 | 공식 URL | 결과 |
|---|---|---|---|
| `graduate/dgist/data.js` | 동일 | 1,944개 동일 | 통과 |
| `graduate/snu/data.js` | 동일 | 2,384개 동일 | 통과 |
| `graduate/kaist/data.js` | 동일 | 6,615개 동일 | 통과 |
| `graduate/postech/data.js` | 동일 | 2,461개 동일 | 통과 |
| `intern/data.js` | 동일 | 329개 동일 | 통과 |

이 비교에는 객체 키, 배열 순서, 교수 순서, 연구 키워드, 점수 관련 값, 매칭 근거와 링크가 포함된다.

## 코드 비교

다음 실행 파일은 수정 전후 바이트 단위로 동일하다.

- `intern/app.js`
- `graduate/dgist/app.js`
- `graduate/snu/app.js`
- `graduate/kaist/app.js`
- `graduate/postech/app.js`
- `analytics.js`
- `privacy-consent.js`
- 기존 HTML 파일 전체

따라서 추천 점수식, intent classifier, 정렬, 더보기, 필터, 만족도와 통계 이벤트 로직은 변경되지 않았다.

## 실제 브라우저 비교

모바일 화면 크기에서 카드 텍스트, 카드 순서, 숨김 상태, 매칭 근거와 링크를 직렬화한 뒤 수정 전후 해시를 비교했다.

### DGIST, 6개

- 바이오센서
- 컴파일러
- 반도체 공정
- 디스플레이
- 전고체전지
- 의료영상 딥러닝

### 서울대학교, 2개

- 3D IC 패키징
- 전고체전지

### KAIST, 1개

- AI 반도체 회로설계

### POSTECH, 1개

- HCI 사용자 경험

### 인턴, 6개

- 반도체 공정
- 배터리
- AI 데이터
- 세메스
- CUop
- 바이오 제약

총 16개 검색에서 결과 직렬화 해시가 수정 전후 동일했다.

## 브라우저 전수 테스트 제한

대용량 학교 데이터 파일을 반복 로딩하는 과정에서 headless Chromium 통신 시간 초과가 발생해 모든 내부 검색어를 브라우저로 전수 실행하지는 못했다. 대신 자동 추출한 내부 검색 필드의 고유 값은 다음과 같았으며, 이 값들을 포함한 비연락처 데이터 구조가 수정 전후 전체 일치함을 확인했다.

- DGIST: 5,645개
- 서울대학교: 7,869개
- KAIST: 14,470개
- POSTECH: 8,648개

모든 임의 자연어 문장을 전수 증명했다는 의미는 아니다. 다만 추천 코드가 그대로이고 연락처 이외의 입력 데이터가 전체 구조 및 순서까지 동일하므로, 일반 연구 분야 검색 결과와 순서가 유지되도록 검증했다.
