## 유지한 정보

- 교수 이름과 필요한 영문 이름
- 학교, 학과, 트랙과 연구실명
- 연구 분야, 연구 키워드와 사실 중심 요약
- 교수 공식 프로필 URL과 연구실 공식 홈페이지 URL
- `positive_queries`, `weak_queries`, `negative_queries`
- 분야 분류, 동의어, 검색 근거, 관련도 점수와 정렬 데이터
- 인턴 기업명, 공고명, 직무, 일정, 지원 자격과 공고 URL
- 개인정보 동의 및 거부 기능
- GoatCounter 계정 주소, pageview 경로와 모든 `lm-` 이벤트

## 변경하지 않은 부분

- 인턴 및 네 학교의 `app.js`
- `analytics.js`
- `privacy-consent.js`
- 모든 기존 HTML 화면
- 검색 점수식, intent classifier, 정렬식과 더보기 구조
- 모바일 및 데스크톱 UI

## 한계

임의의 모든 자연어 검색을 브라우저에서 전수 실행하는 것은 현실적으로 불가능하다. 대신 다음 방식으로 동일성을 검증했다.

1. 수정 전 데이터에서 동일한 연락처 제거 규칙을 적용한 정규화 결과와 수정 후 전체 데이터 구조를 비교했다.
2. 모든 비연락처 필드, 배열 순서, 숫자, 키워드와 URL이 동일함을 확인했다.
3. 추천, 정렬, UI와 통계 코드는 수정 전후 SHA-256이 동일함을 확인했다.
4. 대표 검색어는 실제 브라우저에서 추가 비교했다.

따라서 이메일이나 전화번호 자체를 검색하는 비정상적인 경우를 제외한 일반 연구 분야 및 인턴 검색은 기존 결과를 유지하도록 구성했다.
