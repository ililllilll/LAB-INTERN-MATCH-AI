# LAB MATCH AI 통계 기능 보완 요약

## 기존 기능으로 가능했던 분석

- 서비스 및 학교 선택 비중
- 학교별 검색 실행량
- 결과 0개 및 결과 11개 이상 비율
- 더보기, 인접 분야, 공식 링크 및 공고 링크 클릭량
- 결과 순위 구간별 클릭량

## 이번에 추가한 기능

- 검색어 원문 대신 브라우저 안에서 익명 분야 범주만 분류
- 학교 및 분야별 검색 실패율과 결과 과다 비율 분석
- 검색 결과 아래 `도움이 됐어요`, `아쉬워요` 익명 피드백 추가
- 피드백 질문 표시 횟수도 기록해 응답률 계산 가능
- GoatCounter JSON export를 분석하는 Python 스크립트 추가
- 수정일 기준 전후 비교와 두 비율 검정 지원

## 개인정보 보호

- 검색어 원문, 이름, 학번, 이메일, 전화번호를 전송하지 않음
- 사용자 ID나 자체 추적 ID를 생성하지 않음
- 교수명과 기업명을 이벤트 이름에 기록하지 않음
- 기존 GoatCounter 대시보드와 주소만 사용

## 기존 서비스 유지

- 추천 데이터와 추천 알고리즘은 변경하지 않음
- 기존 pageview 집계와 누적 기록 유지
- `data.js`, `app.js` 분리 구조 유지

## 교수·연구실 데이터 보호 조치

- 만족도 안내 문구를 `선택하신 의견은 더 나은 추천 서비스를 만드는 데 익명으로 활용됩니다.`로 변경
- 학교별 서비스에 비공식 참고 서비스 및 관련도 순위 안내 추가
- `data-policy.html`에 출처, 최소 수집 범위, 평가 아님, 정정·삭제 요청 절차 공개
- `DATA_GOVERNANCE.md`에 공식 출처 우선, 사진·연락처 제외, 사실 중심 요약, 정기 점검 원칙 문서화
- GitHub Issues 기반 정정·삭제 요청 창구 추가

## 공개 데이터 개인정보 최소화

- 연락처 전용 필드 900개 제거
- 긴 원문과 검색 문자열의 이메일 형식 186회 및 전화번호 형식 171회 제거
- 교수, 인턴 레코드 수와 공식 URL 유지
- 추천 알고리즘, UI, 통계 및 개인정보 동의 코드 변경 없음
- 비연락처 데이터 구조 전체 비교와 대표 브라우저 검색 비교 통과

## 분야 중심 통합 연구실 추천 추가

- 첫 화면에 `분야별 연구실 추천` 선택지 추가
- `graduate/field/`에 네 대학 통합 연구실 추천 서비스 추가
- DGIST 158, 서울대 387, KAIST 447, POSTECH 236개, 총 1,228개 데이터 통합
- 큰 연구 분야 12개, 세부 분야 129개 구성
- 로봇과 자율주행, 반도체 공정과 소자 등 유사하지만 다른 분야를 분리
- 교수 카드에 대학 및 소속 학과 표시
- 통합 검색 이벤트와 Python 분석기 반영
