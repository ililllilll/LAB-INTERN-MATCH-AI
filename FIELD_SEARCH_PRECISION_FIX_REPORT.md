# 분야별 연구실 추천 정밀 검색 오류 수정 보고서

## 수정 대상

사용자 점검에서 확인된 다음 네 가지 문제만 수정했다.

1. `UX`가 auxin, flux, tableaux 내부 문자열과 일치하는 문제
2. `식물 면역`처럼 여러 개념을 입력해도 하나만 맞는 연구실이 추천되는 문제
3. `정보보안` 직접 검색은 1명, `보안·암호` 배너는 120명으로 서로 모순되던 문제
4. `자율주행` 검색에 엔진·유체·소프트로봇 등 일반 기계공학 연구실이 대량 포함되던 문제

원본 교수 DB와 학교별 추천 알고리즘은 변경하지 않았다.

## 수정 내용

### 짧은 영문 토큰 경계 검사

`UX`, `HCI`, `AI`, `CT`, `MRI`처럼 3자 이하인 영문 약어는 완전한 토큰으로 존재할 때만 일치하도록 변경했다.

따라서 `UX`는 다음 단어 내부와 더 이상 일치하지 않는다.

- auxin
- flux
- tableaux

### 다중 개념 AND 판정

직접 검색에 두 개 이상의 의미 있는 개념이 있으면 모든 개념이 실제 세부 연구 근거에 존재해야 결과로 인정한다.

예:

- `식물 면역`: 식물 연구와 면역 연구가 모두 확인되는 후보만 표시
- `보안 암호`: 보안과 암호가 모두 확인되는 후보만 표시
- `자율주행 보안`: 자율주행과 보안이 모두 확인되는 후보만 표시

`정보보안`, `사용자 경험`, `자율주행`처럼 하나의 복합 개념으로 굳어진 표현은 하나의 개념으로 처리한다.

### 분야별 정밀 프로필

다음 분야에는 별도의 정밀 판정 규칙을 적용했다.

- HCI·UX
- 정보보안·암호
- 자율주행

넓은 통합 태그, 자동 주입 alias, 다른 분야에 공통으로 붙은 분류 문구는 정밀 판정 근거에서 제외한다. 실제 세부 분야, 연구실명, 공식 요약에 가까운 정보만 사용한다.

### 정보보안 직접 검색과 배너 일치성

- 직접 검색 후보: 34명
- 배너 후보: 33명
- 배너 후보 33명 전원이 직접 검색 후보에도 포함됨

기존의 1명 대 120명 모순은 제거됐다.

## 보존 확인

변경 파일:

- `graduate/field/app.js`

변경하지 않은 항목:

- `graduate/field/data.js` 원본 통합 교수 DB
- DGIST·서울대·KAIST·POSTECH 학교별 DB와 추천 코드
- 인턴 추천 서비스
- 첫 화면 배치
- `analytics.js`
- GoatCounter 이벤트 이름
- 개인정보 동의 및 거부 기능

기존 통계 분석기는 그대로 사용할 수 있다. 검색 결과 수 구간과 공식 링크 클릭 이벤트 이름이 유지되므로, 수정 전후 결과 0개 비율과 검색 후 행동을 같은 방식으로 비교할 수 있다.
